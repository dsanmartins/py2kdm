# Pipeline Configuration

## Purpose

`run_pipeline.py` uses a JSON configuration file to control extraction, architecture recovery and KDM generation.

A typical configuration is stored in `configs/`.

Example:

```bash
python run_pipeline.py --config configs/pymape_hierarchical.json
```

## Configuration structure

A configuration file has the following shape:

```json
{
  "project_name": "pymape_hierarchical",
  "language": "python",

  "input": {
    "source_path": "examples/pymape_hierarchical"
  },

  "outputs": {
    "intermediate_json": "outputs/pymape_hierarchical/python_model.json",
    "architecture_json": "outputs/pymape_hierarchical/python_model.architecture.json",
    "kdm_xmi": "outputs/pymape_hierarchical/model.kdm.xmi"
  },

  "architecture_recovery": {
    "enabled": true,
    "mode": "semi_automatic",
    "target_architecture": "mapek"
  },

  "kdm_generation": {
    "enabled": true,
    "validate": true,
    "input": "architecture_json"
  }
}
```

## Required fields

| Field | Meaning |
|---|---|
| `input.source_path` | Python project to analyze. |
| `outputs.intermediate_json` | JSON generated by the extractor. |
| `outputs.kdm_xmi` | Final KDM XMI output. |

If `outputs.architecture_json` is omitted, the pipeline can derive a default path from the intermediate JSON path.

## Architecture recovery section

```json
"architecture_recovery": {
  "enabled": true,
  "mode": "semi_automatic",
  "target_architecture": "mapek"
}
```

When enabled, the pipeline runs `kdm_architecture_recovery` after extraction.

The recovery stage enriches the intermediate JSON with:

- `architecture_recovery`;
- `structure_model`;
- MAPE-K role suggestions;
- components;
- control loops;
- subsystems;
- containment relationships;
- semantic construction reports.

## KDM generation section

```json
"kdm_generation": {
  "enabled": true,
  "validate": true,
  "input": "architecture_json"
}
```

The `input` field can select the JSON source used by the KDM generator.

Recommended values:

| Value | Meaning |
|---|---|
| `architecture_json` | Use the architecture-enriched JSON. |
| `intermediate_json` | Use only the extractor output. |

When `validate` is `true`, the generator runs JSON-level and KDM-level validation.

## Pipeline flags

The pipeline runner supports:

```bash
python run_pipeline.py --config configs/pymape_hierarchical.json
python run_pipeline.py --config configs/pymape_hierarchical.json --skip-extractor
python run_pipeline.py --config configs/pymape_hierarchical.json --skip-architecture
python run_pipeline.py --config configs/pymape_hierarchical.json --skip-kdm
python run_pipeline.py --config configs/pymape_hierarchical.json --python /path/to/python
```

## Example workflows

Run everything:

```bash
python run_pipeline.py --config configs/pymape_hierarchical.json
```

Only regenerate the KDM from existing JSON:

```bash
python run_pipeline.py \
  --config configs/pymape_hierarchical.json \
  --skip-extractor \
  --skip-architecture
```

Inspect the architecture JSON without generating KDM:

```bash
python run_pipeline.py \
  --config configs/pymape_hierarchical.json \
  --skip-kdm
```

## Output artifacts

Typical outputs are:

```text
outputs/<project>/python_model.json
outputs/<project>/python_model.architecture.json
outputs/<project>/model.kdm.xmi
```
